EAI (Enterprise Application Integration)

Integration framework and architecture

EAI provides messaging and integration between Siebel and external systems. It's powered by a Message Server that routes and transforms data.

ğŸ’¡ EAI Components

  • Message Server: Broker that handles message routing and delivery
  • EAI Adapter: Translator between Siebel and external systems
  • Message: Data packet containing business information
  • Document: Structured XML/JSON representing business data
  • Transport: Protocol for message delivery (HTTP, FILE, JMS)
  • Workflow Process: Orchestrates message flow

Example Code:
function GetCustomerFromExternal(inputPropSet, outputPropSet) {
  var customerId = inputPropSet.GetProperty("CustomerID");
  var httpSvc = GetService("HTTP Outbound Service");
  
  var inProps = new Object();
  inProps.UrlStr = "https://api.external.com/customers/" + customerId;
  inProps.HTTPMethod = "GET";
  
  var outProps = new Object();
  httpSvc.InvokeMethod("DoHttpRequest", inProps, outProps);
  
  if (outProps.StatusCode == "200") {
    outputPropSet.SetProperty("Status", "Success");
    outputPropSet.SetProperty("CustomerData", outProps.Body);
  } else {
    outputPropSet.SetProperty("Status", "Failed");
    outputPropSet.SetProperty("ErrorMsg", outProps.Body);
  }
}

ğŸ“‹ Error Handling in Integration

Try/Catch Pattern:
try {
  var result = InvokeExternalService(...);
  ProcessResult(result);
} catch (e) {
  LogError("Integration failed: " + e.message);
  CreateErrorRecord(e);
  ThrowException(e.message);
}

Retry Pattern:
var maxRetries = 3;
var retryCount = 0;
while (retryCount < maxRetries) {
  try {
    result = InvokeService(...);
    break; // Success
  } catch (e) {
    retryCount++;
    if (retryCount >= maxRetries) throw e;
    WaitSeconds(5); // Wait before retry
  }
}
